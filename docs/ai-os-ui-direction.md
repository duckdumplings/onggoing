# AI OS UI/UX Direction

> 작성: 2026-05-30.
> 목적: "추론 기반 에이전트가 본문인 운영 콘솔"로의 UI/UX 진화 방향을 못 박는다.
> 상위 근거: `docs/design-system/north-star.md`(정체성 5축·레퍼런스·카피 톤). 본 문서는 그 하위 적용 문서.
> 범위: Phase 0~2(기반 정리 → 오케스트레이션 → 메모리). Phase 3~4(자동화 워크플로·멀티에이전트)는 현재 오버스펙으로 제외.
>
> **Living Document**: 레이아웃 재편/도킹 rail 도입 등 구조 변화가 실제 반영되면 §3·§5 갱신.

## 0. 한 문장 방향

> **"에이전트를 모달(이류 시민)에서 → 지도 중심 운영 콘솔의 상시 코파일럿(1급 시민)으로 승격."**
>
> 단, 레퍼런스(카카오모빌리티/티맵/토스/채널톡)는 전부 *지도+패널 운영 어드민*이지 ChatGPT식 풀스크린 챗이 아니다. 지도는 canvas로 유지하고, 에이전트는 그 위의 도킹 코파일럿이 된다.

## 1. 현재 UI의 구조적 한계 (감사 2026-05-29 기반)

- **AI 챗이 모달** → 열고 닫는 부가기능. "추론 기반 제품"인데 에이전트가 본문이 아니다.
- **패널 ↔ 챗 이중 입력** → 같은 경로를 두 군데서 따로 편집(상태 분리). "패널 좁다/챗으로 다 하자" 논쟁의 근본 원인.
- **에이전트 결과가 마크다운 벽** → 숫자/표/지도가 본문이어야 하는데(§정밀) 텍스트로 흘린다.
- 토큰 정의는 견고하나 **사용처 0에 가까움**, 인라인 글래스·이모지·컬러섀도우 잔존.

## 2. 방향성 — North Star 5축 매핑

| From (현재) | To (방향) | 근거 axis / 레퍼런스 |
|---|---|---|
| 챗 = 모달 | **상시 도킹 코파일럿 rail** (지도는 canvas 유지) | 표현(지도가 본문) · 채널톡 도킹 패턴 |
| 패널·챗 이중 상태 | **단일 `RoutePlanDraft` 공유** — 자연어/구조화 두 입력이 같은 지도·상태 편집 | 활기(optimistic) · 카카오모빌리티 |
| 마크다운 답변 | **구조화 결과 객체**(시나리오 카드·출발매트릭스 표·지도 오버레이) | 정밀(.tabular) · 티맵비즈/토스·자비스 |
| "지도에서 보기" 버튼 | 경로 산출 시 **지도 즉시 반영이 기본** | 표현 · 카카오모빌리티 |
| 결과만 노출 | **추론 투명성**: step chips + "왜 이 출발지"(B2) 근거 카드 | 진중 + 신뢰 |
| 기능 진입 산재 | **Cmd-K 옴니박스**(자연어+명령) = OS 입구 | 활기 · (North Star Phase3 → 앞당길 가치) |

## 3. 레퍼런스별 차용 포인트

- **카카오모빌리티 어드민** → 지도 canvas 중심 + 좌측 운영 패널 + 우측 코파일럿. 메인 레이아웃 골격.
- **채널톡 어드민** → 운영 화면에 **대화 rail이 도킹**된 패턴 = 에이전트 rail의 직접 레퍼런스(모달 아님).
- **티맵 비즈 어드민** → 다중 경유지 순서·ETA 표현 = 시나리오/출발매트릭스 결과 카드.
- **토스페이먼츠 / 자비스** → 견적·요금제 표 가독성(.tabular, 단위 정렬) = 견적 카드/문서.
- **Vercel AI SDK "Generative UI"**(도구 결과 → 컴포넌트 매핑) → 마크다운 대신 결과를 컴포넌트로 렌더하는 기술 패턴.
- *(미감 제외, 인터랙션만 차용)* Linear Cmd-K — 명령 팔레트 패턴만.

## 4. 스코프 & 순서 (변화 등급 게이트)

"모달 → 도킹 rail"은 North Star §5 **변화 등급 `obvious`(구조 변화) → 사전 컨펌·단계적**. 한 번에 갈아엎지 않는다.

1. **(noticeable) 결과 객체화 먼저** — 에이전트 답변의 시나리오/매트릭스/근거를 구조화 카드 + 지도 오버레이로. 코드 위험 낮고 **B1(출발지 open-start)과 시너지**(B1의 "왜 이 출발지" 근거가 바로 카드로).
2. **(noticeable) 단일 상태 공유** — `RoutePlanDraft`를 패널·챗 공용 소스로.
3. **(obvious · 컨펌 후) 레이아웃 재편** — 챗 모달 → 도킹 rail, Cmd-K 옴니박스.

> **착수 진입점**: 1번이 B1 작업과 가장 잘 맞물린다. B1으로 출발지 근거가 생기므로, 그 근거를 마크다운이 아니라 카드/지도 오버레이로 보여주는 UI를 함께 잡는다.

## 5. 동반 정리 (리뉴얼과 함께)

패널을 손대는 김에 anti-slop 부채를 함께 청산:
- 인라인 글래스(`bg-white/80 backdrop-blur-2xl`) → `.glass-*` tier 유틸리티
- 운영 UI 이모지(80+곳) → lucide-react 아이콘
- 컬러 섀도우(`shadow-indigo-500/5`) → 토큰 기반 평면 + 강조점에서만 깊이
